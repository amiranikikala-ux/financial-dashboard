## 1. მიზანი + მეთოდი

**მიზანი:** dashboard = ბუღალტრის ნამუშევრის verification ინსტრუმენტი. ყოველ სექციაში 99-100% სიზუსტე.

**მეთოდი:** sprint-by-sprint — ერთი სექცია სრულდება ან წაიშლება, სანამ შემდეგზე გადავდივართ. შემდეგი სექცია იწყება მხოლოდ user-ის ცხადი „გადავიდეთ"-ის შემდეგ.

**კრიტიკული გადაწყვეტილებები (2026-04-28 user პასუხები):**
- შპს ჯეო ფუდთაიმი (ს/კ 400333858) — 4 ფიზიკური ლოკაცია (2 მოქმედი + 2 დაკეტილი), 2 ბანკი (TBC+BOG), POS+სალარო per-store, Megaplus სალარო (per-sale cost + margin pre-computed), 4-5 თანამშრომელი
- 2-ფაზიანი verification window: ზედნადები 2023-2026 (3 წელი, 4 მაღაზია — სრული AP); ბანკი/გაყიდვა/ხარჯი 2026-01-დან
- AP per supplier — ერთიანი view 4-ვე მაღაზიიდან
- დივიდენდი არ აღებულა; სესხი / ლიზინგი არ არსებობს; ყველა ფართი ფიზ.პირისგან ნაქირავები (20% წყაროსთან საშემოსავლო)
- დაარქივება feature — non-supplier (ერთჯერადი) ფირმების display-filter (თანხა უცვლელი); Phase 2 = ფიზიკური ხელით ნაწერი ზედნადები (ბოლოს)

## 3. Sprint cycle (ყოველ სექციას ვრცელდება)

```
1. Scope agree     →  user და AI ცხრილში — რას ვრცავთ, რას არ
2. Data inventory  →  რა გვაქვს, რა აკლია, რა ვითხოვ user-ისგან
3. Spot-check      →  3-5 რეპრეზენტატიული მაგალითი ცხადი წყაროდან
4. Implement       →  კოდი (frontend + backend)
5. Verify          →  source→calc→output proof + breakdown proof
6. User review     →  user ხედავს, ცხადად ამბობს „გადავიდეთ" ან „ჩავასწოროთ"
```

**არცერთ ნაბიჯზე გადახტომა.** Spot-check-ი data-ს არასაკმარისობას ცხადყოფს → ვბრუნდები 2-ზე.

## 4. პრინციპები (ყოველ სექციას ვრცელდება)

| პრინციპი | რას ნიშნავს |
|---|---|
| **Source-first** | ციფრი „PROOFED"-ია, როცა source→calc→output სამივე მტკიცდება. სხვა შემთხვევაში: „გაკეთებულია, მაგრამ ბოლომდე დამტკიცებული არ არის" |
| **Verification, not preparation** | dashboard ბუღალტრის შემცვლელი არ არის. side-by-side: ბუღალტრის ციფრი / ჩვენი ციფრი / სხვაობა / ახსნა |
| **No silent gap** | ცარიელი slot — ცხადად „უცნობი" / „მონაცემი აკლია" / „candidate ვერ ამოვიღეთ" |
| **Breakdown proof** | aggregate proof ≠ breakdown proof. ყოველი per-store / per-month / per-product ცალკე უნდა მტკიცდეს |
| **Plain language** | UI ტექსტი ქართულად — „რა უნდა მომხდარიყო, რა მოხდა, რა შეიცვალა". ჟარგონი მინიმუმამდე |
