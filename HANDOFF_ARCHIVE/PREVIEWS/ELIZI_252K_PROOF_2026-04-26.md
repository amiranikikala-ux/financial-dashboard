## Cancelled status inflation — FIXED 2026-04-27

User authorized fix immediately after the proof landed. Pipeline now skips
`სტატუსი == "გაუქმებული"` rows BEFORE the composite-key dedup, with a
tally surfaced in `bundle["overall"]` mirroring `duplicate_rows_dropped`.

### Code change (`dashboard_pipeline/imported_products.py`)

* Added module-level constant `CANCELLED_STATUS_KA = "გაუქმებული"`.
* Inserted filter immediately after status extraction in the row loop:
  ```python
  if status == CANCELLED_STATUS_KA:
      cancelled_rows_dropped += 1
      cancelled_amount_dropped += amount_ge
      continue
  ```
* Surfaced two new keys in `overall`: `cancelled_rows_dropped` +
  `cancelled_amount_dropped_ge`.
* Filter runs BEFORE dedup so cancelled rows do not poison the
  `seen_row_keys` set (which would risk dropping legitimate active rows
  sharing the same composite key).

### Regression tests (`tests/test_imported_products_cancelled_filter.py`)

5 new pytest tests pinning:
1. Cancelled rows excluded from row_count + total_amount_ge + supplier stats.
2. Cancelled status no longer appears in `by_status` accumulator.
3. Cancelled rows do NOT poison the dedup seen-set (active row sharing
   composite key with a cancelled row is preserved).
4. Zero-cancelled corpus keeps `cancelled_rows_dropped == 0`.
5. Cancelled amount tally uses raw face value (independent of dedup logic).

`pytest tests/test_imported_products_cancelled_filter.py` → 5/5 PASS.
Full suite: 2,229 PASS / 3 FAIL (3 pre-existing phase4b3 doc tests, baseline
unchanged from before the fix). No new failures.

Note on the two cancelled tallies:

* `−109,744.78 ₾` is the **net reduction in `total_amount_ge`** (matches
  the pre-fix `by_status["გაუქმებული"].total_ge` exactly: 109,744.78).
* `146,814.40 ₾` is the **face-value sum of every cancelled row dropped**,
  including duplicates among themselves. Pre-fix, those duplicates would
  have been collapsed by the composite-key dedup; post-fix they are all
  filtered before dedup runs. The `−667` shift in `duplicate_rows_dropped`
  is exactly the count of cancelled-rows-that-were-also-duplicates that
  used to be tallied as duplicate-drops.

Both numbers are correct and useful: net reduction tells you the real
inflation removed, face-value tally tells you how many raw cancelled
records the source contains.
