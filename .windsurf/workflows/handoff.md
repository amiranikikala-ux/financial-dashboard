## Handoff Workflow — ახალი ჩატისთვის მომზადება

### როდის გააქტიურდეს:
- კონტექსტი ~100K token-ს უახლოვდება (ბევრი tool call, დიდი ფაილები წაკითხული)
- მიმდინარე ამოცანა დასრულდა ან ლოგიკურ წერტილამდე მივიდა
- მომხმარებელი ითხოვს handoff-ს

### ნაბიჯები:

1. **გაჩერდი ლოგიკურ წერტილზე** — არ დატოვო კოდი გატეხილ მდგომარეობაში. თუ ედითი დაწყებულია, დაასრულე.

2. **გაუშვი ტესტები** — დარწმუნდი რომ ყველაფერი მუშაობს:
// turbo
```
cd rs-dashboard && npx vite build
```
// turbo
```
cd rs-dashboard && npx playwright test
```

3. **განაახლე HANDOFF.md** — ჩაამატე:
   - სესიის ნომერი და თარიღი
   - რა ფაილები შეიცვალა/შეიქმნა
   - რა გაკეთდა (მოკლედ)
   - ტესტების სტატუსი (unit + E2E + build)
   - Known Issues თუ ახალი გაჩნდა

4. **განაახლე PLAN.md** — მონიშნე დასრულებული ამოცანები `[x]`-ით, დაამატე ახალი თუ საჭიროა.

5. **დაწერე მომხმარებელს:**
   ```
   ⚠️ კონტექსტი ივსება. HANDOFF.md და PLAN.md განახლებულია.
   ახალ ჩატში დაწერე: "წაიკითხე HANDOFF.md და PLAN.md და განაგრძე"
   
   დარჩენილი ამოცანები: [ჩამოთვალე]
   ```

### წესები:
- არასოდეს დატოვო კოდი broken state-ში
- HANDOFF.md-ში ყოველთვის ჩაწერე ბოლო სესიის ფაილების სია
- ტესტების რაოდენობა ყოველთვის მიუთითე (unit + E2E)
- თუ რაღაც ვერ დასრულდა, ჩაწერე "დარჩენილი" სექციაში
